# Система управления университетской библиотекой

## Описание
Консольное приложение для автоматизации работы университетской библиотеки. Реализовано на C# с использованием принципов объектно-ориентированного программирования.

## Функционал
- Управление библиотечным фондом (книги, учебники, справочники)
- Выдача и возврат материалов
- Поиск по каталогу (по автору, названию)
- Статистика библиотеки
- Ведение учета читателей

## Архитектура и принципы ООП

### Принципы SOLID
1. **Single Responsibility Principle (SRP)**
   - `LibraryService` - только бизнес-логика
   - `ItemRepository` - только работа с данными
   - `LibraryItem` - только представление библиотечного материала

2. **Open/Closed Principle (OCP)**
   - Абстрактный класс `LibraryItem` позволяет добавлять новые типы материалов без изменения существующего кода
   - Интерфейсы `ILendable`, `ILibraryService` обеспечивают расширяемость

3. **Liskov Substitution Principle (LSP)**
   - `Book`, `Textbook`, `Reference` могут использоваться вместо `LibraryItem` без нарушения логики

4. **Interface Segregation Principle (ISP)**
   - Разделение интерфейсов: `ILendable` для выдачи, `ILibraryService` для операций, `IItemRepository` для данных

5. **Dependency Inversion Principle (DIP)**
   - Внедрение зависимостей через конструкторы
   - Зависимость от абстракций, а не конкретных реализаций

### Паттерны проектирования
1. **Repository Pattern** - `ItemRepository` инкапсулирует доступ к данным
2. **Service Layer** - `LibraryService` отделяет бизнес-логику от представления
3. **Strategy Pattern** - различные алгоритмы поиска через интерфейсы
4. **Template Method** - абстрактный метод `DisplayInfo()` в `LibraryItem`

### Наследование и полиморфизм
- Иерархия классов: `LibraryItem` ← `Book` ← (`Textbook`, `Reference`)
- Виртуальные и абстрактные методы
- Переопределение `DisplayInfo()` в наследниках
